# Universal Proxy (WSS & SSH)

Универсальный SOCKS5 прокси-клиент и сервер, который способен туннелировать весь трафик либо через защищенное WebSocket (WSS) соединение, либо через SSH-туннель. Система разработана для обхода DPI блокировок на основе архитектуры `tg-ws-proxy`, но работает с любыми адресами и протоколами поверх TCP.

## Режимы работы

В файле `config.json` можно выбрать режим работы клиента (параметр `"mode"`):
1. **`wss`**: Трафик маскируется под обычное 웹-соединение. Для работы в этом режиме необходимо запустить `server.py` на вашем удаленном сервере (VPS).
2. **`ssh`**: Трафик направляется через SSH-туннель. Работает "из коробки" с любым сервером, на котором включен SSH. Запускать `server.py` на VPS в этом случае не нужно.

## Конфигурация (`config.json`)

```json
{
  "mode": "wss", 
  "server_ws_host": "127.0.0.1",
  "server_ws_port": 8443,
  "ssh_host": "127.0.0.1",
  "ssh_port": 22,
  "ssh_user": "root",
  "ssh_key": "/path/to/key",
  "local_socks5_port": 1080,
  "bypass_domains": [".ru", "localhost"],
  "bypass_ips": ["127.0.0.0/8"]
}
```
- **bypass_domains / bypass_ips**: Домены и IP-адреса, запросы к которым пойдут напрямую от вашего провайдера (в обход прокси).

## Быстрый старт (Режим WSS)

1. **Сервер (на VPS):**
   ```bash
   python server.py --host 0.0.0.0 --port 443
   ```
2. **Клиент (Локально):**
   Настройте `config.json` (укажите `"mode": "wss"` и IP вашего сервера в `"server_ws_host"`).
   ```bash
   python client.py
   ```
3. Настройте ваш браузер, Telegram или системный SOCKS5 на `127.0.0.1:1080`.

## Быстрый старт (Режим SSH)

1. **Сервер (на VPS):**
   Ничего запускать не нужно, если на сервере работает стандартный `sshd`.
2. **Клиент (Локально):**
   Настройте `config.json` (укажите `"mode": "ssh"`, IP, логин и путь к приватному ключу SSH).
   ```bash
   python client.py
   ```

## Глобальный роутинг (перехват всего трафика ОС)

SOCKS5 по умолчанию перехватывает трафик только тех приложений, где он явно прописан. Чтобы перенаправить *весь* трафик вашего компьютера через `universal_proxy`, используйте виртуальный TUN-адаптер:
- **Windows**: Proxifier, tun2socks, Nekoray.
- Укажите в этих программах локальный SOCKS5 `127.0.0.1:1080`.
