# Изменения Universal Proxy

## Введение
Создан проект `universal_proxy` на основе реверс-инжиниринга `tg-ws-proxy`.

## 1. Архитектура и Протоколы
- **Проблема:** Оригинальный прокси работал только для подсетей Telegram и зависел от их специфичного протокола MTProto и серверов (kws.web.telegram.org).
- **Решение:** Реализована клиент-серверная архитектура. Прокси теперь может обрабатывать любой SOCKS5 трафик.
- **Технологии:** `asyncio`, нативные сокеты, `asyncssh` для альтернативного туннелирования.
- **Улучшения:** Поддержка двойного режима (`wss` и `ssh`), обход (bypass) локальных и российских доменов/IP напрямую.
